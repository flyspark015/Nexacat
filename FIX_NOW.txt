ERROR YOU'RE SEEING:
  ❌ Error getting admin conversation: FirebaseError: [code=permission-denied]
  ❌ Error loading conversation: FirebaseError: [code=permission-denied]

WHY:
  Firestore security rules haven't been deployed to Firebase yet.

FIX (2 MINUTES):

STEP 1: OPEN FIREBASE CONSOLE
  • Go to: https://console.firebase.google.com
  • Click your FlySpark project
  • Click "Firestore Database" (left sidebar)
  • Click "Rules" tab (at top)

STEP 2: DELETE OLD RULES
  • Click in the rules editor
  • Press Ctrl+A (select all)
  • Press Delete

STEP 3: COPY NEW RULES
  • Open this file: /COPY_THESE_RULES.txt
  • Press Ctrl+A (select all)
  • Press Ctrl+C (copy)

STEP 4: PASTE NEW RULES
  • Go back to Firebase Console
  • Click in the empty rules editor
  • Press Ctrl+V (paste)

STEP 5: PUBLISH
  • Click the blue "Publish" button (top-right)
  • Wait for "Rules published successfully" message
  • This takes 10-30 seconds

STEP 6: TEST
  • Go back to FlySpark app
  • Press F5 to refresh
  • Click AI Assistant button (purple bot icon)
  • Should work! ✅

TROUBLESHOOTING:

Still not working?
  1. Wait 1 minute for rules to propagate
  2. Refresh your app again (F5)
  3. Try the "Run Checks" button in Admin Settings

Need to verify your admin role?
  1. Firebase Console → Firestore Database → Data
  2. Click "users" collection
  3. Find your user document
  4. Check that "role" field = "admin"
  5. If not, change it to "admin" and save

Clear browser cache:
  1. Press Ctrl+Shift+Delete
  2. Select "Cached images and files"
  3. Click "Clear data"
  4. Refresh page
